# 戦略 — Figmaナビ

## 売りたいもの

**Figma プラグイン「Figmaナビ」の有料サブスクリプション**。
AI がデザインを生成する。デザインシステム準拠・チームのルール入力対応が強み。
価格は **¥980 / 約50命令** など、初心者も買える低価格帯で構成（無料枠あり・3プラン）。

## 運用フォーマット: ニュース中心

**「毎朝チェックしないと乗り遅れる」FOMO を作る**ために、コンテンツはニュース性を主軸とする。Tips のみだと習慣化されず、既存200人を超えて伸ばすのが難しい。

| 曜日 | コンテンツ型 | 優先度 |
|---|---|---|
| 月〜金 | **毎朝のデザインニュース**（Figma/プラグイン/業界/AI×デザイン） | Primary |
| 土・日 | **Tips / Before-After / ポートフォリオ事例** | Secondary |

週末の Tips はニュースで触れた話題の実践編（「今週発表された Figma の新機能、早速試した」等）として接続すると、平日ニュース→週末深掘りの循環が生まれる。

## ペルソナ（優先順）

### Primary: デザイナー志望者・駆け出しデザイナー
- 既存 YouTube デザイン講座チャンネルに **約200人のサブスクライバー** がいる
- その層の延長として、スキルアップ中・転職準備中・独学者が中心
- Figma は触れるが、ゼロからの作例作りに時間がかかる
- ポートフォリオ / 実案件想定の作例 / LP作成 などの実用シーンを抱えている
- ¥980 なら学習投資として手が出る
- **既存の信頼関係があるため、day-1 の転換源として最重要**

### Secondary: 個人デザイナー / フリーランス
- 量をこなしたい。下書きの時短が直接売上に効く
- Layermate 等と比較検討する層
- 価格で勝てる余地がある

### Tertiary: 非デザイナー職（PM / エンジニア / 創業者）
- 「デザイナー雇えない」「外注の時間も予算もない」
- LP、資料、管理画面、プロトタイプを自分で作る必要がある
- 業務時間短縮＝対価を払う動機が強い

## 差別化ポイント

1. **価格** — ¥980/約50命令の低価格帯。Layermate 等よりわかりやすく安い
2. **デザインシステム / ルール入力可能** — 将来の法人展開の布石
3. **チーム利用を前提とした設計**
4. **日本語ネイティブ対応**（UI / サポート / 事例動画）
5. **教える人が作っている** — デザイン講座運営者による、学習者の詰まりどころを理解したプラグイン

## 探索方針（Discovery Strategy）

### 英語優先原則

**英語ソースを先に見る**。デザインツール系・業界動向系のニュースは、英語が日本語より 1〜3 日早い。英語で拾って日本語で噛み砕く方が速報性・独自性で勝る。
日本語ソースは補完（日本企業のリデザイン、日本語コミュニティ声など）。

### 推奨ソース（シード）

**英語（優先）**:
- Figma changelog / Figma Blog — 公式リリース情報
- Figma Community — 新着プラグイン
- UX Collective (Medium) — 業界論考
- Smashing Magazine — UX/UI 実践
- Designer News / Dribbble Trending
- Hacker News "design" keyword — 技術者層の議論
- Product Hunt — 新プロダクト、AI ツール
- Awwwards — 受賞作品
- X（英語）の著名デザイナー（Khoi Vinh, Mike Buzzard 等）
- r/Design, r/UI_Design, r/web_design

**日本語（バックアップ）**:
- cocoda（デザイナーコミュニティ）
- note デザイン記事
- X 日本語デザイナー界隈
- Designer News 日本語記事

### Discovery Methods（カテゴリタグ）

| タグ | 説明 |
|---|---|
| `news-en` | 英語のデザイン/テック業界メディア |
| `news-ja` | 日本語メディア（cocoda, note等） |
| `official-src` | Figma changelog / 公式ブログ / リリースノート |
| `figma-community` | Figma Community の新着プラグイン |
| `social-en` | X / Reddit / HN の英語デザイナー界隈 |
| `social-ja` | X 日本語デザイナー界隈 |
| `product-launches` | Product Hunt / Awwwards / Dribbble Trending |
| `expert-blog` | 個人デザイナー・業界有識者の発信 |

### 80/20 Exploit/Explore

- 過去14日で視聴率上位3位の method は 80% の頻度で使う（exploit）
- 残り 20% は**新しい method を試す**（explore）
- Explore でヒットした method は次サイクルで exploit 側に昇格

### Meta PDCA（探し方の PDCA）

毎日のルーチンで以下を分析:

1. 直近 14 日の `performance-history.json` から **method → 平均 views / プラグインインストール** テーブルを作成
2. TOP 3 / WORST 3 method を特定
3. 「今日はどの method で探すか」を 80/20 ルールで決める
4. 新しい method アイデアは PDCA レポートの「Method 提案」セクションに記録
5. **WORST method は 2 週連続なら一旦凍結**

## NG パターン

| NG | 理由 |
|---|---|
| プロ向けの高度な議論 | Primary 層が置いていかれる |
| 「AIがデザイナーの仕事を奪う」系 | 学習中の視聴者を不安にさせる。**学習者の味方として振る舞う**のが方針 |
| プラグインの機能説明メイン | 興味ない層が離脱。必ず「学習者の課題」から入る |
| 英語/日本語の中途半端な混在 | JPチャンネルはJP主。ただし**入力情報は英語**、出力が日本語 |
| 「AIに任せれば考えなくていい」系訴求 | 学習者向けに不誠実。AIは**補助輪**として位置づける |
| **1週間以上前のニュースを新着扱い** | 鮮度が命。`freshness_hours > 168` は原則 NG |
| **出典不明ニュース** | 信用毀損。ソース URL を必ず記録・明示する |

## 英語展開方針

- YouTube: 英語字幕必須、動画内テキストは日英併記 or 英語版を別チャンネルで運用
- Instagram: キャプションの冒頭は英語、2行目以降日本語
- 判断: まず JP で 500 インストール到達したら EN 展開を本格化

```

  ↓ 新動画（毎朝デザインニュース）

  ↓ プロフィールへ

  ↓ Figma Community URL
Figma Community ページ（プラグインインストール）
  ↓ インストール
無料プランで試す
  ↓ 命令数の制限に到達

  ↓
```
